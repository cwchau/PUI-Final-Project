Body Music Generator

Final Term Project --- May. 2018 (3 weeks)

Context
Design and create an interactive application using Python for the introductory programming course, Fundamentals of Programming and Computer Science (15-112) at Carnegie Mellon University.
Key Concepts:
  • Game design
  • Programming
  • Interactivity
  • User testing

Tools Used:
  • Python
  • Microsoft Visual Studio
  • Microsoft Kinect
  • pythonOSC, pyKinect, pygame
  • Max Cycling '74

Background
I created "Body Music Generator" for my final term project for the introductory programming course, Fundamentals of Programming and Computer Science (15-112) at Carnegie Mellon University.

As a music lover, I wanted to create a project that allows people to explore and have fun. I got inspiration from Dalcroze Eurythmics, a development approach to teaching music through your body and the natural connection people have to music and sound.

The "Body Music Generator" is very close to my heart as it marked the first time I created a complete and working interactive application with code.


This project marks the beginning of my passion and interest in the intersection between people, art, and technology.

Features
"Body Music Generator" was created in Python and utilizes the Microsoft Kinect via pyKinect and pygame. In addition, I used an Open Sound Control module, pythonOSC, so that I could connect the joint-tracking data from the Kinect to sound variables with Max, a visual programming language used commonly for sound and music.

Key features of this project include:

  • 'Learn Mode' including lessons on basic music theory (dynamics, note explortion, etc.)
  • 'Game Mode' including minigames such as note identifier and song-player
  • 'Freestyle Mode' including sound manipulation
  • 2-player mode
Volume

Volume is measured by the vertical distance between the user's hands. The closer their hands are together, the quieter the sound which mirrors real life where bringing your limbs and self tighter together makes you feel "small". Conversely, stretching out your arms vertically makes you seem taller and intimidating, like a loud sound.

Pitch

Pitch is controlled by the horizontal distance between the user's hands. The closer they are, the closer the pitch is to the original sound. Stretching your arms out wide increasingly distorts the sound.

Lessons Learned
This project was a lot of fun to create. It took some time to program the core functionality because I wanted to include lots of features. One important topic that I caught a glimpse of in this assignment, was the importance of user testing.

While testing the 'Freestyle Mode' application with a few friends, they were unsure if the system was working or not. To address this issue, I created the two key visuals indicated in the images above.

For volume, there is a red bar on the right that adjust its height according to the volume. For pitch, there is a colored bar at the top that shifts from reddish-pink to a purple-blue to signify the pitch shift.

Without the feedback from users, I would have missed creating visuals, leaving my users confused and uncertain.
